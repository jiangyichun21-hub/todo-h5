# Vibe Coding 体验总结

> 从零开始用 AI 编程做一个多用户待办事项应用的完整记录

---

## 一、项目概述

**目标**：用自然语言描述需求，让 AI 编程工具帮我写代码，做一个支持多用户的待办事项应用。

**最终成果**：
- 一个完整的 Web 应用，支持用户注册登录
- 每个用户可以管理自己的待办事项
- 数据存储在云端，刷新不丢失
- 多用户数据完全隔离

---

## 二、技术栈

| 技术 | 用途 |
|------|------|
| Next.js 16 | 前端框架 |
| TypeScript | 类型安全 |
| Tailwind CSS | 页面样式 |
| Supabase | 数据库 + 用户认证 |
| Claude Code | AI 编程助手 |
| GitHub | 代码托管 |
| Vercel | 部署平台 |

---

## 三、开发过程回顾

### Day 1：环境搭建
- 用 `npx create-next-app` 一键创建项目
- 安装了 TypeScript 和 Tailwind CSS
- AI 自动处理了所有配置，我只需要等

### Day 2：搭建页面骨架
- 用一句话描述想要的界面，Claude Code 直接生成了代码
- 标题、输入框、按钮、列表区域一次到位
- 刷新浏览器就能看到效果

### Day 3-4：核心功能
- 添加、删除、完成、筛选，四个功能分两天完成
- 每次输入一段自然语言描述，AI 就帮我实现
- 中间遇到 Claude Code API 配额用完的问题，换成我直接改代码

### Day 5-7：数据库 + 用户认证 + 多用户隔离
- 这是最复杂的部分，涉及 Supabase 接入
- 注册 Supabase、建表、配置 API Key 需要我手动操作
- 代码部分全部交给 Claude Code，它自动生成了：
  - Supabase 客户端配置
  - 登录注册页面
  - 用户认证逻辑
  - 数据隔离（每个用户只能看自己的待办）
- 测试了两个账号，确实互相看不到对方的数据

### Day 8：样式美化
- 跳过了，默认样式已经够用

### Day 9：数据验证
- 测试了刷新页面数据还在
- 测试了关闭浏览器重新打开数据也在
- 全部通过

### Day 10：部署上线
- GitHub 仓库创建和推送顺利
- Vercel 部署遇到两个坑：
  1. `package-lock.json` 里记录了阿里内网 npm 源，Vercel 访问不了，重新生成解决
  2. `npm install` 报错是因为 Node.js 版本问题，指定 22.x 解决
- 最终 Vercel 部署成功，但国内需要 VPN 才能访问
- 尝试 Cloudflare Pages 解决国内访问问题，但 next-on-pages 不支持 Next.js 16，失败

## 四、Claude Code 使用体验

### AI 理解意图的准确度
- **简单 UI 描述**：非常准确，说「加个输入框和按钮」就加对了
- **功能逻辑描述**：大部分准确，偶尔需要补充说明
- **复杂架构设计**：需要拆分成小步骤，一次性说太多会漏掉细节

### 平均多少轮对话完成一个功能
- 简单功能（加个按钮、改个颜色）：1 轮
- 中等功能（添加待办、删除待办）：1-2 轮
- 复杂功能（用户认证、数据隔离）：3-5 轮
- 平均下来一个功能 2 轮左右

### 代码质量
- 生成的代码结构清晰，能用
- TypeScript 类型定义完整
- 样式用了 Tailwind CSS，类名规范
- 偶尔会有小 Bug（比如变量名写错），但指出后能快速修正

### 遇到的问题
1. **API 配额限制**：Claude Code 用的模型有调用次数限制，用完了要等或换模型
2. **内网 npm 源**：我本地用的是阿里内网源，生成的 `package-lock.json` Vercel 部署不了
3. **Next.js 版本太新**：Cloudflare Pages 的部署工具不支持 Next.js 16

## 五、总结

### AI 擅长的场景
- **从零搭建项目**：环境配置、依赖安装、项目结构，AI 一键搞定
- **UI 页面生成**：描述界面长什么样，AI 直接出代码
- **标准功能实现**：增删改查、登录注册，这些常见功能 AI 非常熟练
- **代码调试**：报错了把错误信息丢给 AI，它能快速定位问题

### AI 不擅长的场景
- **第三方服务接入**：像 Supabase 注册、建表这种需要网页操作的，AI 帮不上忙
- **部署环境问题**：npm 源、Node.js 版本兼容，需要人工排查
- **跨平台兼容**：Cloudflare Pages 不支持 Next.js 16，AI 也没提前预警

### 爽在哪
- 不用自己写代码，只需要描述想要什么
- 从 0 到 1 做出一个完整应用，只用了十几天
- 以前要学几个月的前端知识，现在几天就体验了一遍

### 卡在哪
- API 配额用完要等，打断节奏
- 部署环节踩了不少坑，花的时间比写代码还多
- 国内访问 Vercel 部署的应用需要 VPN，不够完美

### 对 Vibe Coding 的整体评价

**Vibe Coding 不是替代程序员，而是降低编程门槛。**

作为一个没有前端经验的产品经理，我能在十几天里做出一个支持多用户的 Web 应用，这在以前是不可想象的。AI 编程工具让我把精力集中在「我想要什么」而不是「代码怎么写」上。

**适合谁**：
- 想快速验证想法的创业者
- 想做个人项目的产品经理
- 想学编程但不知道从何开始的初学者

**不适合谁**：
- 需要高性能、高并发的生产级应用（还是需要专业程序员）
- 需要深度定制和优化的场景

**一句话总结**：Vibe Coding 让「会描述需求」比「会写代码」更重要。
